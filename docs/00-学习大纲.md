# Objective-C iOS 开发学习路线

> 目标：从零构建一个完整的 iOS 应用（待办事项 App — "OCTodo"）
> 受众：有 Web 开发经验、iOS 基础薄弱的开发者
> 方式：每个阶段 = 知识讲解 + 代码实战 + 文档沉淀

---

## 项目选型：为什么做一个 Todo App？

一个 Todo App 虽然简单，但能覆盖 iOS 开发的核心知识点：
- UI 布局（UITableView / UINavigationController）
- 数据模型与持久化（NSUserDefaults / Core Data）
- 网络请求（NSURLSession）
- 生命周期与内存管理
- 设计模式（MVC / Delegate / Block / Notification）

## 每节课的产出

1. **文档**：`docs/XX-课程名.md` — 知识点讲解 + Web 类比
2. **代码**：对应的 OC 源文件，带详细注释
3. **练习**：每节课末尾的小练习，巩固知识

---

## 学习原则

- **先理解，再动手**：每个概念先用 Web 开发的思维类比，再看 OC 怎么做
- **代码即文档**：所有代码都有注释，解释"为什么"而不只是"是什么"
- **循序渐进**：不跳步骤，每一步都在前一步的基础上构建
- **实战驱动**：语法学完就用到项目里，不脱离实际

## 当前安排复盘（截至第 03 节）

当前安排整体方向是正确的，但有三个可优化点：

1. `第 05 节` 同时放 `Category/Extension/KVC/KVO`，知识密度过高，且 KVC/KVO 缺少业务场景承托。  
2. `AutoLayout` 放在 `UIViewController 生命周期`前，初学者容易只会“摆控件”，不理解“页面什么时候做什么”。  
3. `NSUserDefaults → Core Data` 放在同一节，跨度大，容易造成“都懂一点但都不扎实”。

基于以上问题，后续采用“先跑通，再升级”的策略：先完成可见结果，再补原理和进阶。
