# 第一课：Objective-C 基础语法

> 用 Web 开发的视角理解 OC

---

## 1. OC 是什么？

Objective-C（简称 OC）是 C 语言的超集，在 C 的基础上加了一套面向对象的语法。

**Web 类比：**
- C 语言 ≈ 原生 JavaScript（基础能力）
- Objective-C ≈ TypeScript（在 JS 基础上加了类型系统和面向对象）
- 但 OC 的面向对象语法来自 Smalltalk，和 Java/JS 的风格很不一样

## 2. 文件结构

| OC 文件 | 作用 | Web 对应 |
|---------|------|---------|
| `.h` (Header) | 声明文件：声明类有哪些属性和方法 | `.d.ts` 类型声明文件 |
| `.m` (Implementation) | 实现文件：写具体逻辑 | `.ts` / `.js` 实现文件 |

**为什么要分两个文件？**
- C/OC 是编译型语言，编译器需要提前知道"有哪些类、方法"
- JS 是解释型语言，运行时才去找，所以不需要分开
- 可以把 `.h` 理解为"目录"，`.m` 理解为"正文"

## 3. 基本语法对照

### 3.1 导入模块

```objc
// OC: #import 防止重复导入（类似 #pragma once）
#import <UIKit/UIKit.h>      // 系统框架，用尖括号
#import "MyClass.h"           // 自己的文件，用引号
```

```javascript
// JS/TS 对应
import UIKit from 'UIKit';       // 系统模块
import MyClass from './MyClass'; // 自己的模块
```

### 3.2 定义一个类

```objc
// ====== Person.h（声明） ======
#import <Foundation/Foundation.h>

@interface Person : NSObject  // Person 继承自 NSObject（万物之父）

@property (nonatomic, copy) NSString *name;  // 属性声明
@property (nonatomic, assign) NSInteger age; // 基本类型用 assign

- (void)sayHello;              // 实例方法声明（- 开头）
+ (Person *)personWithName:(NSString *)name; // 类方法声明（+ 开头）

@end

// ====== Person.m（实现） ======
#import "Person.h"

@implementation Person

- (void)sayHello {
    // NSLog 类似 console.log
    // %@ 是对象占位符，类似 JS 的 ${变量}
    NSLog(@"你好，我是 %@，今年 %ld 岁", self.name, (long)self.age);
}

+ (Person *)personWithName:(NSString *)name {
    Person *p = [[Person alloc] init];
    p.name = name;
    return p;
}

@end
```

```typescript
// TypeScript 对应
class Person {
  name: string;
  age: number;

  sayHello(): void {
    console.log(`你好，我是 ${this.name}，今年 ${this.age} 岁`);
  }

  static personWithName(name: string): Person {
    const p = new Person();
    p.name = name;
    return p;
  }
}
```

### 3.3 关键语法差异速查表

| 概念 | JavaScript / TypeScript | Objective-C |
|------|------------------------|-------------|
| 创建对象 | `new Person()` | `[[Person alloc] init]` |
| 调用方法 | `obj.method(arg)` | `[obj method:arg]` |
| 字符串 | `"hello"` / `` `hello` `` | `@"hello"` |
| 打印日志 | `console.log()` | `NSLog()` |
| 空值 | `null` / `undefined` | `nil` |
| self/this | `this` | `self` |
| 继承 | `class A extends B` | `@interface A : B` |
| 数组 | `[1, 2, 3]` | `@[@1, @2, @3]` |
| 字典 | `{key: "value"}` | `@{@"key": @"value"}` |
| 布尔值 | `true` / `false` | `YES` / `NO` |

## 4. 指针 `*` — 你一定看到了满屏的星号

项目代码里到处都有 `*`，比如：

```objc
UIWindow *window;
NSString *name;
HomeViewController *homeVC;
```

**这是什么？**
OC 中所有对象类型的变量都是"指针"——它存的不是对象本身，而是对象在内存中的地址。

**Web 类比：**
```javascript
// JS 里，对象变量本质上也是"引用"（类似指针）
const obj = { name: "小明" };  // obj 存的是对象的引用，不是对象本身
const obj2 = obj;              // obj2 和 obj 指向同一个对象
```

```objc
// OC 里，* 显式地告诉你"这是一个指针/引用"
NSString *name = @"小明";      // name 是一个指向 NSString 对象的指针
NSString *name2 = name;        // name2 和 name 指向同一个字符串
```

**简单记忆：** 在 OC 中，只要是对象类型（NSString、UIView、自定义类等），声明时都要加 `*`。基本类型（int、BOOL、NSInteger）不需要。

```objc
NSString *name;       // 对象类型 → 要 *
UIView *view;         // 对象类型 → 要 *
NSInteger age;        // 基本类型 → 不要 *
BOOL isFinished;      // 基本类型 → 不要 *
```

## 5. 方法调用语法（最大的不同！）

OC 的方法调用使用 **方括号 + 冒号分隔参数**，这是最让人不习惯的地方：

```objc
// OC 的方法调用：[接收者 方法名:参数1 参数标签2:参数2]

// 无参数
[person sayHello];
// JS: person.sayHello()

// 一个参数
[person setName:@"小明"];
// JS: person.setName("小明")

// 多个参数（OC 的特色：每个参数都有标签！）
[person setName:@"小明" age:25];
// JS: person.set("小明", 25)  ← JS 里你看不出 25 是什么
// OC 的参数标签让代码自带文档，一看就知道每个参数是什么
```

**理解方法签名：**

```objc
// 方法签名 = 方法名 + 所有参数标签连起来
// 下面这个方法的完整签名是：insertObject:atIndex:
- (void)insertObject:(id)object atIndex:(NSUInteger)index;

// 对比 JS：
// insertObject(object, index) ← 签名就是 insertObject
```

## 5. 属性（Property）关键字

```objc
@property (nonatomic, strong) NSArray *items;
//         ^^^^^^^^^  ^^^^^^
//         线程安全    内存管理
```

| 关键字 | 含义 | 什么时候用 |
|--------|------|-----------|
| `strong` | 强引用，"我拥有它" | 对象类型的属性（默认） |
| `weak` | 弱引用，"我只是看看" | 避免循环引用（delegate 常用） |
| `copy` | 拷贝一份 | NSString、Block |
| `assign` | 直接赋值 | 基本类型（int、BOOL 等） |
| `nonatomic` | 不加线程锁 | 几乎所有属性都用这个 |
| `atomic` | 加线程锁（默认） | 极少用，性能差 |

**Web 类比：**
- `strong` ≈ JS 的普通引用 `const obj = otherObj`
- `weak` ≈ JS 的 WeakRef（ES2021）
- `copy` ≈ `const str = otherStr.slice()` 防止被外部修改

## 6. 嵌套调用与链式访问 — 读懂项目里的"套娃"代码

项目代码中有这样的写法，初次看会很懵：

```objc
// main.m 中的这行：
UIApplicationMain(argc, argv, nil, NSStringFromClass([AppDelegate class]));

// AppDelegate.m 中的这行：
self.window = [[UIWindow alloc] initWithFrame:[UIScreen mainScreen].bounds];
```

### 6.1 嵌套消息发送（从里往外读）

```objc
NSStringFromClass([AppDelegate class])
//                 ^^^^^^^^^^^^^^^^^ 先执行：获取 AppDelegate 的 Class 对象
// ^^^^^^^^^^^^^^^^                  再执行：把 Class 转成字符串 @"AppDelegate"
```

**Web 类比：**
```javascript
// 就像 JS 的嵌套函数调用，从里往外读
String(AppDelegate.constructor)  // 先拿到构造函数，再转字符串
```

```objc
[[UIWindow alloc] initWithFrame:[UIScreen mainScreen].bounds]
//                               ^^^^^^^^^^^^^^^^^^^^^^ 第1步：获取屏幕对象
//                               ^^^^^^^^^^^^^^^^^^^^^^^^^^^ 第2步：.bounds 拿屏幕尺寸
// ^^^^^^^^^^^^^^^^ 第3步：alloc 分配内存
// ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^ 第4步：用屏幕尺寸初始化窗口
```

**Web 类比：**
```javascript
// 相当于
new UIWindow({ frame: UIScreen.mainScreen().bounds })
```

### 6.2 点语法 vs 方括号语法

OC 有两种访问方式，容易混淆：

```objc
// 点语法：访问属性（getter/setter）
self.view.backgroundColor       // 读取属性
self.title = @"OCTodo";         // 设置属性

// 方括号语法：调用方法
[UIColor whiteColor];           // 调用类方法
[self setupWelcomeLabel];       // 调用实例方法
```

**规则：**
- **属性** 用点语法 `.`（类似 JS 的 `obj.prop`）
- **方法** 用方括号 `[]`（类似 JS 的 `obj.method()`）
- 实际上点语法是方括号的语法糖：`self.title` 等价于 `[self title]`

```objc
// 这两种写法完全等价：
self.view.backgroundColor = [UIColor whiteColor];
[[self view] setBackgroundColor:[UIColor whiteColor]];
// 但第一种更简洁，实际开发中属性都用点语法
```

### 6.3 `[super viewDidLoad]` — 调用父类方法

```objc
- (void)viewDidLoad {
    [super viewDidLoad];  // 必须先调用父类的实现
    // 然后写自己的逻辑...
}
```

**Web 类比：**
```javascript
class HomeViewController extends UIViewController {
  viewDidLoad() {
    super.viewDidLoad();  // JS 的 super 调用
    // 自己的逻辑...
  }
}
```

`super` 指向父类，`self` 指向当前实例。就像 JS 的 `super` 和 `this`。

## 7. `@autoreleasepool` — main.m 里的那个大括号

```objc
int main(int argc, char * argv[]) {
    @autoreleasepool {
        return UIApplicationMain(argc, argv, nil, ...);
    }
}
```

这是 OC 的内存管理机制。暂时不用深究原理（第二课会详细讲），只需要知道：
- 它创建了一个"自动释放池"，App 运行期间产生的临时对象会被自动清理
- **Web 类比：** JS 有垃圾回收器自动做这件事，OC 需要显式声明这个池子
- 这行代码是模板代码，每个 iOS App 都有，永远不用改

## 8. 枚举常量 — 那些全大写的值

项目代码中有很多看起来像常量的东西：

```objc
welcomeLabel.textAlignment = NSTextAlignmentCenter;
welcomeLabel.font = [UIFont systemFontOfSize:24 weight:UIFontWeightBold];
```

这些是 **枚举值（Enum）**，类似 TypeScript 的枚举：

```typescript
// TypeScript 对应
enum NSTextAlignment {
  Left,
  Center,   // ← NSTextAlignmentCenter
  Right,
}
label.textAlignment = NSTextAlignment.Center;
```

**常见枚举命名规律：** `类型前缀 + 具体值`
- `NSTextAlignment` + `Center` → `NSTextAlignmentCenter`
- `UIFontWeight` + `Bold` → `UIFontWeightBold`
- 看到全大写开头的长名字，基本就是枚举常量

## 9. NSLog — OC 的 console.log

```objc
NSLog(@"Hello");                          // console.log("Hello")
NSLog(@"名字: %@", name);                 // console.log(`名字: ${name}`)
NSLog(@"年龄: %ld", (long)age);           // console.log(`年龄: ${age}`)
NSLog(@"价格: %.2f", price);              // console.log(`价格: ${price.toFixed(2)}`)
NSLog(@"是否完成: %@", done ? @"YES" : @"NO");
```

**常用格式符：**
- `%@` — 对象（NSString、NSArray 等）
- `%d` / `%ld` — 整数
- `%f` — 浮点数
- `%@` — 会自动调用对象的 `description` 方法（类似 JS 的 `toString()`）

## 10. 协议（Protocol） — AppDelegate.h 里的尖括号

```objc
@interface AppDelegate : UIResponder <UIApplicationDelegate>
//                       ^^^^^^^^^   ^^^^^^^^^^^^^^^^^^^^^^^
//                       继承自谁      遵守什么协议
```

**Web 类比：**
```typescript
// TypeScript
class AppDelegate extends UIResponder implements UIApplicationDelegate {
  // 必须实现协议要求的方法
}
```

- 协议 = 一组方法的约定（类似 TS 的 interface）
- `<UIApplicationDelegate>` 表示"我承诺实现这个协议规定的方法"
- 所以 AppDelegate.m 里才有 `didFinishLaunchingWithOptions` 等方法——这些都是协议要求的

后续第四课会深入讲 Protocol，这里先知道尖括号 `< >` 是遵守协议的意思。

## 12. 练习

在继续下一课之前，试着回答这些问题：

1. `[[Person alloc] init]` 对应 JS 里的什么？
2. `@property (nonatomic, copy) NSString *name;` 为什么 NSString 要用 `copy` 而不是 `strong`？
3. OC 方法 `- (void)setName:(NSString *)name age:(NSInteger)age` 的完整方法签名是什么？
4. `#import` 和 JS 的 `import` 有什么区别？
5. 为什么 `NSString *name` 要加 `*`，而 `NSInteger age` 不用加？
6. `[[UIWindow alloc] initWithFrame:[UIScreen mainScreen].bounds]` 的执行顺序是什么？（从哪里开始读？）
7. `self.title = @"OCTodo"` 和 `[self setTitle:@"OCTodo"]` 有什么关系？
8. `<UIApplicationDelegate>` 中的尖括号表示什么意思？用 TypeScript 怎么表达？

> **答案提示：**
> 1. `new Person()`
> 2. 因为 NSString 有一个可变子类 NSMutableString，如果用 strong，外部修改了原字符串，你的属性也会变；copy 确保你拿到的是不可变的副本
> 3. `setName:age:`
> 4. `#import` 是预处理指令，在编译前就把内容复制过来；JS 的 `import` 是模块系统在运行时加载
> 5. `*` 表示指针/引用。对象类型（NSString、UIView 等）都是指针，需要 `*`；基本类型（NSInteger、BOOL）直接存值，不需要 `*`
> 6. 从最里面往外读：① `[UIScreen mainScreen]` 获取屏幕对象 → ② `.bounds` 获取屏幕尺寸 → ③ `[UIWindow alloc]` 分配内存 → ④ `initWithFrame:` 用屏幕尺寸初始化窗口
> 7. 完全等价。点语法是方括号调用的语法糖，`self.title = x` 会自动转成 `[self setTitle:x]`
> 8. 遵守协议（Protocol），类似 TypeScript 的 `class AppDelegate implements UIApplicationDelegate`
