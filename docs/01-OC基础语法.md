# 第一课：Objective-C 基础语法

> 用 Web 开发的视角理解 OC

---

## 1. OC 是什么？

Objective-C（简称 OC）是 C 语言的超集，在 C 的基础上加了一套面向对象的语法。

**Web 类比：**
- C 语言 ≈ 原生 JavaScript（基础能力）
- Objective-C ≈ TypeScript（在 JS 基础上加了类型系统和面向对象）
- 但 OC 的面向对象语法来自 Smalltalk，和 Java/JS 的风格很不一样

## 2. 文件结构

| OC 文件 | 作用 | Web 对应 |
|---------|------|---------|
| `.h` (Header) | 声明文件：声明类有哪些属性和方法 | `.d.ts` 类型声明文件 |
| `.m` (Implementation) | 实现文件：写具体逻辑 | `.ts` / `.js` 实现文件 |

**为什么要分两个文件？**
- C/OC 是编译型语言，编译器需要提前知道"有哪些类、方法"
- JS 是解释型语言，运行时才去找，所以不需要分开
- 可以把 `.h` 理解为"目录"，`.m` 理解为"正文"

## 3. 基本语法对照

### 3.1 导入模块

```objc
// OC: #import 防止重复导入（类似 #pragma once）
#import <UIKit/UIKit.h>      // 系统框架，用尖括号
#import "MyClass.h"           // 自己的文件，用引号
```

```javascript
// JS/TS 对应
import UIKit from 'UIKit';       // 系统模块
import MyClass from './MyClass'; // 自己的模块
```

### 3.2 定义一个类

```objc
// ====== Person.h（声明） ======
#import <Foundation/Foundation.h>

@interface Person : NSObject  // Person 继承自 NSObject（万物之父）

@property (nonatomic, copy) NSString *name;  // 属性声明
@property (nonatomic, assign) NSInteger age; // 基本类型用 assign

- (void)sayHello;              // 实例方法声明（- 开头）
+ (Person *)personWithName:(NSString *)name; // 类方法声明（+ 开头）

@end

// ====== Person.m（实现） ======
#import "Person.h"

@implementation Person

- (void)sayHello {
    // NSLog 类似 console.log
    // %@ 是对象占位符，类似 JS 的 ${变量}
    NSLog(@"你好，我是 %@，今年 %ld 岁", self.name, (long)self.age);
}

+ (Person *)personWithName:(NSString *)name {
    Person *p = [[Person alloc] init];
    p.name = name;
    return p;
}

@end
```

```typescript
// TypeScript 对应
class Person {
  name: string;
  age: number;

  sayHello(): void {
    console.log(`你好，我是 ${this.name}，今年 ${this.age} 岁`);
  }

  static personWithName(name: string): Person {
    const p = new Person();
    p.name = name;
    return p;
  }
}
```

### 3.3 关键语法差异速查表

| 概念 | JavaScript / TypeScript | Objective-C |
|------|------------------------|-------------|
| 创建对象 | `new Person()` | `[[Person alloc] init]` |
| 调用方法 | `obj.method(arg)` | `[obj method:arg]` |
| 字符串 | `"hello"` / `` `hello` `` | `@"hello"` |
| 打印日志 | `console.log()` | `NSLog()` |
| 空值 | `null` / `undefined` | `nil` |
| self/this | `this` | `self` |
| 继承 | `class A extends B` | `@interface A : B` |
| 数组 | `[1, 2, 3]` | `@[@1, @2, @3]` |
| 字典 | `{key: "value"}` | `@{@"key": @"value"}` |
| 布尔值 | `true` / `false` | `YES` / `NO` |

## 4. 方法调用语法（最大的不同！）

OC 的方法调用使用 **方括号 + 冒号分隔参数**，这是最让人不习惯的地方：

```objc
// OC 的方法调用：[接收者 方法名:参数1 参数标签2:参数2]

// 无参数
[person sayHello];
// JS: person.sayHello()

// 一个参数
[person setName:@"小明"];
// JS: person.setName("小明")

// 多个参数（OC 的特色：每个参数都有标签！）
[person setName:@"小明" age:25];
// JS: person.set("小明", 25)  ← JS 里你看不出 25 是什么
// OC 的参数标签让代码自带文档，一看就知道每个参数是什么
```

**理解方法签名：**

```objc
// 方法签名 = 方法名 + 所有参数标签连起来
// 下面这个方法的完整签名是：insertObject:atIndex:
- (void)insertObject:(id)object atIndex:(NSUInteger)index;

// 对比 JS：
// insertObject(object, index) ← 签名就是 insertObject
```

## 5. 属性（Property）关键字

```objc
@property (nonatomic, strong) NSArray *items;
//         ^^^^^^^^^  ^^^^^^
//         线程安全    内存管理
```

| 关键字 | 含义 | 什么时候用 |
|--------|------|-----------|
| `strong` | 强引用，"我拥有它" | 对象类型的属性（默认） |
| `weak` | 弱引用，"我只是看看" | 避免循环引用（delegate 常用） |
| `copy` | 拷贝一份 | NSString、Block |
| `assign` | 直接赋值 | 基本类型（int、BOOL 等） |
| `nonatomic` | 不加线程锁 | 几乎所有属性都用这个 |
| `atomic` | 加线程锁（默认） | 极少用，性能差 |

**Web 类比：**
- `strong` ≈ JS 的普通引用 `const obj = otherObj`
- `weak` ≈ JS 的 WeakRef（ES2021）
- `copy` ≈ `const str = otherStr.slice()` 防止被外部修改

## 6. NSLog — OC 的 console.log

```objc
NSLog(@"Hello");                          // console.log("Hello")
NSLog(@"名字: %@", name);                 // console.log(`名字: ${name}`)
NSLog(@"年龄: %ld", (long)age);           // console.log(`年龄: ${age}`)
NSLog(@"价格: %.2f", price);              // console.log(`价格: ${price.toFixed(2)}`)
NSLog(@"是否完成: %@", done ? @"YES" : @"NO");
```

**常用格式符：**
- `%@` — 对象（NSString、NSArray 等）
- `%d` / `%ld` — 整数
- `%f` — 浮点数
- `%@` — 会自动调用对象的 `description` 方法（类似 JS 的 `toString()`）

## 8. 练习

在继续下一课之前，试着回答这些问题：

1. `[[Person alloc] init]` 对应 JS 里的什么？
2. `@property (nonatomic, copy) NSString *name;` 为什么 NSString 要用 `copy` 而不是 `strong`？
3. OC 方法 `- (void)setName:(NSString *)name age:(NSInteger)age` 的完整方法签名是什么？
4. `#import` 和 JS 的 `import` 有什么区别？

> **答案提示：**
> 1. `new Person()`
> 2. 因为 NSString 有一个可变子类 NSMutableString，如果用 strong，外部修改了原字符串，你的属性也会变；copy 确保你拿到的是不可变的副本
> 3. `setName:age:`
> 4. `#import` 是预处理指令，在编译前就把内容复制过来；JS 的 `import` 是模块系统在运行时加载
