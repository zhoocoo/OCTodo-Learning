# 第十课：UINavigationController、页面跳转与数据回传

> 实现多页面交互：从首页跳转到"添加待办"页面，完成后回传数据。

---

## 1. UINavigationController 是什么？

UINavigationController 管理一个**页面栈**，提供 push（进入新页面）和 pop（返回上一页）的能力。

**Web 类比：** React Router / Vue Router 的页面栈。浏览器的"前进"和"后退"就是 push 和 pop。

```
导航栈（从底到顶）：
┌──────────────────┐
│  AddTodoVC       │  ← 栈顶（当前显示）
├──────────────────┤
│  HomeVC          │  ← 栈底（被覆盖，还在内存中）
└──────────────────┘
```

我们在 AppDelegate 中已经创建了导航控制器：
```objc
UINavigationController *nav = [[UINavigationController alloc]
                               initWithRootViewController:homeVC];
```

## 2. 页面跳转方式

### 2.1 Push（压栈）— 最常用

```objc
// 从当前页面跳到新页面
AddTodoViewController *addVC = [[AddTodoViewController alloc] init];
[self.navigationController pushViewController:addVC animated:YES];
// animated:YES 带动画（从右侧滑入）

// 系统自动提供返回按钮（左上角 < 按钮）
```

**Web 类比：**
```javascript
router.push('/add-todo');    // React Router
this.$router.push('/add');   // Vue Router
```

### 2.2 Pop（出栈）— 返回

```objc
// 返回上一页
[self.navigationController popViewControllerAnimated:YES];

// 返回到根页面（跳过中间所有页面）
[self.navigationController popToRootViewControllerAnimated:YES];
```

**Web 类比：**
```javascript
router.back();       // 返回上一页
history.go(-2);      // 返回两页
```

### 2.3 Present（模态弹出）

```objc
// 从底部弹出一个新页面（遮盖当前页面）
AddTodoViewController *addVC = [[AddTodoViewController alloc] init];
[self presentViewController:addVC animated:YES completion:nil];

// 关闭模态页面
[self dismissViewControllerAnimated:YES completion:nil];
```

**Web 类比：** 类似弹出一个 Modal 对话框。

### Push vs Present

| | Push | Present |
|--|------|---------|
| 动画 | 从右滑入 | 从底弹出 |
| 导航栏 | 有（带返回按钮） | 默认没有 |
| 适用场景 | 层级导航（列表→详情） | 独立任务（添加、设置） |
| 返回方式 | pop / 返回按钮 | dismiss |

## 3. 页面间传值

### 3.1 正向传值（A → B）— 属性传值

跳转前直接设置目标页面的属性：

```objc
// HomeVC 跳转到 DetailVC，传递一个 TodoItem
DetailViewController *detailVC = [[DetailViewController alloc] init];
detailVC.todoItem = self.todoItems[indexPath.row];  // 直接赋值
[self.navigationController pushViewController:detailVC animated:YES];
```

**Web 类比：**
```jsx
// React: 通过 props 传值
<DetailPage todoItem={selectedItem} />
// 或通过路由参数
router.push({ path: '/detail', query: { id: item.id } })
```

### 3.2 反向传值（B → A）— 三种方式

新页面完成任务后，需要把结果传回上一个页面。三种常用方式：

#### 方式一：Delegate（本课重点）

最经典的 iOS 回传方式，在第四课已经学过原理，现在实战。

#### 方式二：Block 回调

```objc
// AddTodoVC 定义一个 Block 属性
@property (nonatomic, copy) void (^onTodoAdded)(NSString *title);

// HomeVC 跳转时设置 Block
AddTodoViewController *addVC = [[AddTodoViewController alloc] init];
addVC.onTodoAdded = ^(NSString *title) {
    // 收到新标题，添加到列表
    [self.todoItems addObject:[TodoItem itemWithTitle:title]];
    [self refreshUI];
};
[self.navigationController pushViewController:addVC animated:YES];
```

#### 方式三：NSNotificationCenter（广播）

```objc
// 发送通知（B 页面）
[[NSNotificationCenter defaultCenter] postNotificationName:@"TodoAdded"
                                                    object:nil
                                                  userInfo:@{@"title": title}];

// 监听通知（A 页面）
[[NSNotificationCenter defaultCenter] addObserver:self
                                         selector:@selector(handleTodoAdded:)
                                             name:@"TodoAdded"
                                           object:nil];
```

#### 三种方式对比

| 方式 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **Delegate** | 一对一，多事件 | 结构清晰，编译检查 | 代码量多 |
| **Block** | 一对一，单回调 | 简洁 | 注意循环引用 |
| **Notification** | 一对多，跨层级 | 解耦 | 难追踪，易滥用 |

## 4. 导航栏自定义

```objc
// 设置标题
self.title = @"添加待办";

// 自定义右侧按钮
self.navigationItem.rightBarButtonItem =
    [[UIBarButtonItem alloc] initWithTitle:@"保存"
                                    style:UIBarButtonItemStyleDone
                                   target:self
                                   action:@selector(saveButtonTapped)];

// 自定义左侧按钮（替代默认返回按钮）
self.navigationItem.leftBarButtonItem =
    [[UIBarButtonItem alloc] initWithTitle:@"取消"
                                    style:UIBarButtonItemStylePlain
                                   target:self
                                   action:@selector(cancelButtonTapped)];
```

## 5. 实战：创建"添加待办"页面

我们用 **Delegate** 方式实现完整的跳转和回传流程：

1. HomeVC 点击"添加"按钮 → push 到 AddTodoVC
2. AddTodoVC 输入标题，点"保存"
3. 通过 delegate 回传标题给 HomeVC
4. HomeVC 添加新 TodoItem，刷新列表，AddTodoVC 自动 pop 回来

## 6. 练习

1. `push` 和 `present` 的区别是什么？什么场景用哪个？

2. 为什么反向传值不能用"属性传值"（B 直接设置 A 的属性）？

3. 如果 AddTodoVC 的 delegate 属性用了 strong，会发生什么？

4. Block 回传和 Delegate 回传，你会在什么场景下选择哪个？

> **答案提示：**
> 1. push 是导航栈内的层级跳转（有返回按钮），present 是模态弹出（独立任务）。列表→详情用 push，新建/设置用 present
> 2. B 不应该知道 A 的具体类型（耦合太紧），而且 B 持有 A 的强引用会造成循环引用
> 3. HomeVC strong 持有 AddTodoVC（push 时导航栈持有），AddTodoVC 又 strong 持有 HomeVC（delegate），形成循环引用
> 4. 单个简单回调用 Block（如"添加完成"），多个回调或需要协议约束用 Delegate
